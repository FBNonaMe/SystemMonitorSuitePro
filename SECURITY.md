# Security Policy

## Supported Versions

The following versions of **SystemMonitorSuitePro** currently receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

Only the latest stable release is actively maintained and receives security patches.

---

## Reporting a Vulnerability

If you discover a security vulnerability in **SystemMonitorSuitePro**, please report it responsibly.

Do **not** open a public GitHub issue for security vulnerabilities.

Instead, report the issue using one of the following methods:

Email: **[ddosovh014@gmail.com](mailto:ddosovh014@gmail.com**

Please include the following information:

* Description of the vulnerability
* Steps to reproduce the issue
* Affected version of the software
* Screenshots or proof-of-concept if available

---

## Response Policy

After receiving a vulnerability report:

1. A confirmation will usually be sent within **48 hours**.
2. The issue will be investigated and validated.
3. If confirmed, a security fix will be prepared and released.
4. The reporter may be credited in the security advisory.

If the report is not considered a security vulnerability, an explanation will be provided.

---

## Responsible Disclosure

Please avoid publicly disclosing the vulnerability until a fix has been released.
This helps protect users of the project.

Thank you for helping improve the security of **SystemMonitorSuitePro**.
